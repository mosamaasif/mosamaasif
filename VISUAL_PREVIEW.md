## Theme Switching

The profile automatically switches between dark and light modes using the HTML `<picture>` element:

```html
<picture>
  <source media="(prefers-color-scheme: dark)" srcset="assets/dark_mode.svg">
  <source media="(prefers-color-scheme: light)" srcset="assets/light_mode.svg">
  <img alt="Terminal-styled profile" src="assets/dark_mode.svg">
</picture>
```

**How it works:**
1. Browser checks user's system theme preference
2. If dark mode → loads `dark_mode.svg`
3. If light mode → loads `light_mode.svg`
4. Falls back to dark mode if preference not detected

- Padding: 20px all around

- Right column: 45% width (~450px)
- Column gap: 20px

**Text Positioning:**
- Left column starts at x=10
- Right column starts at x=570
- Top margin: 30px
- Section spacing: 1.5× line height

## Color Accessibility

Both themes have been designed for optimal readability:

**Dark Mode Contrast:**
- Background to foreground: 15.7:1 (AAA)
- Background to yellow: 12.4:1 (AAA)
- Background to blue: 7.3:1 (AA)

**Light Mode Contrast:**
- Background to foreground: 15.7:1 (AAA)
- Background to yellow: 8.9:1 (AAA)
- Background to blue: 7.3:1 (AA)

All color combinations meet WCAG AA standards for normal text.

## Responsive Behavior

While the SVG has fixed dimensions, GitHub automatically scales it for different screen sizes:

**Desktop (1200px+):**
- Full size display (1000×550px)
- Both columns fully visible
- Optimal reading experience

**Tablet (768px-1199px):**
- Scaled down proportionally
- Maintains aspect ratio
- Still readable

**Mobile (320px-767px):**
- Significantly scaled down
- May require horizontal scroll
- ASCII art still recognizable

## Animation Possibilities (Future)

The SVG format supports animations that could be added:

```svg
<!-- Typing effect for text -->
<animate attributeName="opacity" from="0" to="1" dur="0.5s"/>

<!-- Pulse effect for stats -->
<animate attributeName="fill-opacity" values="1;0.7;1" dur="2s" repeatCount="indefinite"/>

<!-- Fade-in for ASCII art -->
<animate attributeName="opacity" from="0" to="1" begin="0.5s" dur="1s"/>
```

Currently not implemented to keep file size small and avoid distraction.
